# DeepThink Style

> AI Output Style — 风格套件 · v1.0

---

## 风格定位

**DeepThink** 是一套以学术咨询报告为灵感的视觉风格。

设计基调：纸张质感、克制的墨色系、精准的信息层级、适合长文阅读。推荐用于分析报告、研究结论、深度调研等内容较多的文档，内容较少时同样可以使用。

---

## 视觉语言

### 色彩语义

| 颜色 | 语义角色 |
|------|----------|
| 暖白 `#fbfbf8` | 页面底色，纸张感 |
| 主色绿 `#1f6f54` | 重点结论、正向信息、强调 |
| 警示琥珀 `#9a5a12` | 分歧、不确定、风险 |
| 链接蓝 `#275f8f` | 链接、次要信息 |
| 墨色系 `#161616 / #303030 / #656565` | 标题 / 正文 / 辅助文字 |

绿色 = 已确认的、正向的。琥珀色 = 需要注意的、存在争议的。这个映射在整套风格里保持一致。

## 使用方式

把目标模板的链接或内容提供给 AI，附上你要美化的内容：

```
请读取这个 Style 模板：
[report.html 或 report.md 的链接或内容]

把以下内容用这套风格渲染：
[你的 AI 输出内容]
```

模板文件的顶部注释里有完整的组件用法说明，AI 读取后即可直接使用。

**指定模板：**

- `template:report` 或 `template:report.html` → HTML 模板（默认）
- `template:report.md` → Markdown 模板

**多模板同时渲染：**

```
templates:
- report.html
- report.md
```

## 文件结构

```
deepthink/
├── README.md              ← 本文件（面向人类）
└── templates/
    ├── report.html        ← HTML 模板（视觉表达优先）
    └── report.md          ← Markdown 模板（阅读节奏优先）
```

---

## 模板选择

| 模板 | 适合场景 | 渲染依赖 |
|------|---------|---------|
| `report.html` | 需要可视化、可打印为 PDF、可在浏览器分享 | 浏览器 |
| `report.md` | GitHub README、文档站、邮件、纯文本归档、PR 描述 | 任意 Markdown 渲染器 |

两份模板共用同一套视觉语义（绿色 = 结论、琥珀 = 风险、深绿 = 约束），
内容可在两者间无损迁移。

**默认模板：** `report.html`

---

## 保真原则（契约声明）

DeepThink 是**视觉风格**，不是**内容编辑器**。

DeepThink 承诺：**只换皮，不改肉。**

- 原文段落 1:1 出现在输出中
- 不删除、不改写、不压缩、不摘要
- 仅添加结构性元信息：章节标题、表头、组件 label

> 详细的执行约束与组件禁忌对照表，见 `templates/report.html` 与 `templates/report.md` 头部注释——模板是保真规则的权威来源。
